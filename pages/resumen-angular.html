Resumen Rápido: Angular

Referencia rápida de comandos CLI, directivas y conceptos clave

Comandos Angular CLI más utilizados

Angular CLI es la herramienta de línea de comandos para crear y gestionar proyectos Angular.

Comando Descripción Ejemplo/Opciones
ng new nombre-app Crea un nuevo proyecto Angular ng new mi-app --routing --style=scss
ng serve Ejecuta el servidor de desarrollo ng serve -o (abre el navegador automáticamente)
ng generate component Genera un nuevo componente ng g c components/header
ng generate service Genera un nuevo servicio ng g s services/data
ng generate module Genera un nuevo módulo ng g m shared
ng build Compila la aplicación para producción ng build --prod
ng test Ejecuta los tests unitarios ng test --code-coverage
ng lint Verifica el código con ESLint ng lint --fix (corrige automáticamente)

Decoradores principales

Los decoradores son funciones que modifican clases, propiedades o métodos en TypeScript/Angular.

Decorador Dónde se usa Qué hace
@Component Clase componente Define un componente con selector, template y estilos
@NgModule Clase módulo Define un módulo que agrupa componentes, servicios, etc.
@Injectable Clase servicio Permite inyectar el servicio en otros componentes/servicios
@Input() Propiedad componente Permite recibir datos del componente padre
@Output() Propiedad componente Permite emitir eventos al componente padre
@ViewChild Propiedad componente Obtiene referencia a un elemento hijo en el template
@HostListener Método componente Escucha eventos del elemento host del componente

Directivas principales

Las directivas modifican el comportamiento o apariencia de elementos en el DOM.

Directiva Tipo Qué hace Ejemplo
*ngIf Estructural Muestra u oculta elemento según condición <div *ngIf="isVisible">
*ngFor Estructural Repite elemento para cada item de un array <li *ngFor="let item of items">
*ngSwitch Estructural Muestra uno de varios elementos según valor <div [ngSwitch]="color">
[ngClass] Atributo Añade/quita clases CSS dinámicamente [ngClass]="{'active': isActive}"
[ngStyle] Atributo Aplica estilos CSS dinámicamente [ngStyle]="{'color': textColor}"
[(ngModel)] Atributo Data binding bidireccional (two-way) [(ngModel)]="username"

Tipos de Data Binding

Sintaxis Tipo Dirección Ejemplo
{{ valor }} Interpolación Componente → Vista <h1>{{ titulo }}</h1>
[propiedad] Property Binding Componente → Vista <img [src]="imagenUrl">
(evento) Event Binding Vista → Componente <button (click)="guardar()">
[(ngModel)] Two-way Binding Bidireccional <input [(ngModel)]="nombre">

Lifecycle Hooks (Ciclo de vida)

Métodos que se ejecutan en momentos específicos del ciclo de vida de un componente.

Hook Cuándo se ejecuta Uso común
ngOnInit() Después de crear el componente Inicializar datos, llamar servicios (el más usado)
ngOnChanges() Cuando cambian propiedades @Input Reaccionar a cambios en inputs del padre
ngDoCheck() En cada detección de cambios Detección personalizada de cambios
ngAfterViewInit() Después de inicializar la vista Acceder a elementos del DOM con @ViewChild
ngOnDestroy() Antes de destruir el componente Limpiar suscripciones, timers, listeners

Operadores RxJS más usados

RxJS es la librería de programación reactiva que usa Angular para manejar datos asíncronos.

Operador Qué hace Cuándo usarlo
map() Transforma cada valor emitido Modificar datos que vienen del servidor
filter() Filtra valores según condición Obtener solo valores que cumplen criterio
tap() Ejecuta código sin modificar el valor Logging, debugging, efectos secundarios
switchMap() Cambia a un nuevo Observable Llamadas HTTP secuenciales
catchError() Maneja errores del Observable Gestionar errores de peticiones HTTP
debounceTime() Espera tiempo antes de emitir Búsquedas en tiempo real (esperar que usuario termine de escribir)
take(n) Toma solo los primeros n valores Limitar número de emisiones
subscribe() Se suscribe al Observable Recibir los valores emitidos (SIEMPRE necesario)

Router - Navegación

Concepto/Método Descripción Ejemplo
RouterModule Módulo de routing de Angular RouterModule.forRoot(routes)
Routes Array de configuración de rutas {path: 'home', component: HomeComponent}
router.navigate() Navega programáticamente this.router.navigate(['/home'])
routerLink Directiva para enlaces de navegación <a routerLink="/home">Inicio</a>
ActivatedRoute Servicio para obtener info de la ruta actual route.params.subscribe(params => ...)
:id Parámetro de ruta {path: 'user/:id', component: UserComponent}
** Ruta comodín (wildcard) para 404 {path: '**', component: NotFoundComponent}

HttpClient - Peticiones HTTP

Método Qué hace Ejemplo
http.get(url) Petición GET para obtener datos http.get<User[]>('/api/users')
http.post(url, body) Petición POST para crear http.post('/api/users', userData)
http.put(url, body) Petición PUT para actualizar http.put('/api/users/1', userData)
http.delete(url) Petición DELETE para eliminar http.delete('/api/users/1')

Formularios

Tipo Características Cuándo usarlo
Template-driven Basado en directivas en el template (ngModel) Formularios simples y rápidos
Reactive Forms Basado en programación en el componente (FormControl, FormGroup) Formularios complejos con validación avanzada

Consejo importante

Angular tiene muchos conceptos, pero una vez entiendes el flujo de datos (data binding), componentes y servicios, todo lo demás encaja naturalmente.

Practica creando componentes pequeños y reutilizables. El patrón más común: componente muestra datos → servicio obtiene datos del servidor → RxJS maneja la asincronía.
