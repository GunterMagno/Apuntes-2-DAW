6. Networking

Comunicación entre contenedores

Introducción

Las redes Docker permiten que los contenedores se comuniquen entre sí de forma segura. Por defecto, Docker crea una red bridge donde todos los contenedores pueden conectarse.

Tipos de Redes

  • bridge: Red por defecto. Contenedores en el mismo host
  • host: Usa la red del host directamente
  • none: Sin red (aislado)
  • overlay: Para Docker Swarm (múltiples hosts)

Crear y Gestionar Redes

# Listar redes
docker network ls

# Crear red personalizada
docker network create mi-red

# Inspeccionar red
docker network inspect mi-red

# Eliminar red
docker network rm mi-red

# Eliminar redes no usadas
docker network prune

Conectar Contenedores

Los contenedores en la misma red pueden comunicarse usando sus nombres como hostname.

# Crear red
docker network create app-network

# Ejecutar MongoDB en la red
docker run -d \
  --name mongodb \
  --network app-network \
  mongo:6

# Ejecutar aplicación que se conecta a MongoDB
docker run -d \
  --name mi-app \
  --network app-network \
  -e MONGO_URL=mongodb://mongodb:27017/mydb \
  mi-app-image

# La app puede conectarse usando "mongodb" como hostname

DNS Interno

Docker proporciona un servidor DNS interno. Los contenedores pueden resolverse por nombre.

# En Docker Compose, se crea automáticamente una red
version: '3.8'
services:
  web:
    image: nginx
  api:
    image: node-app
    environment:
      - WEB_HOST=web  # Puede usar "web" como hostname
  database:
    image: postgres
    
# Los tres servicios están en la misma red y pueden comunicarse

Ejemplo Práctico

# Desde el contenedor "api", puedes hacer:
curl http://web:80
# O conectarte a la base de datos:
psql -h database -U postgres
