5. Volúmenes

Persistencia de datos

¿Qué son los Volúmenes?

Los volúmenes permiten persistir datos fuera del ciclo de vida del contenedor. Sin volúmenes, cuando eliminas un contenedor, pierdes todos sus datos.

Problema: Tienes una base de datos en un contenedor. Si lo eliminas, pierdes todos los datos.

Solución: Usa un volumen para almacenar los datos fuera del contenedor.

Tipos de Volúmenes

  • Named Volumes: Gestionados por Docker (recomendado)
  • Bind Mounts: Montar carpeta del host en el contenedor
  • tmpfs Mounts: Almacenamiento temporal en memoria

Bind Mounts

Montar una carpeta de tu máquina dentro del contenedor. Útil para desarrollo.

# Montar carpeta actual en /app del contenedor
docker run -v $(pwd):/app mi-app

# En Windows PowerShell
docker run -v ${PWD}:/app mi-app

# Ruta absoluta
docker run -v /home/user/proyecto:/app mi-app

# Solo lectura
docker run -v $(pwd):/app:ro mi-app

Cuidado: Los cambios en tu máquina se reflejan en el contenedor y viceversa.

Named Volumes

Volúmenes gestionados por Docker. Mejor para producción.

# Crear volumen
docker volume create mi-volumen

# Usar volumen
docker run -v mi-volumen:/data mi-app

# En Docker Compose
version: '3.8'
services:
  database:
    image: postgres
    volumes:
      - db-data:/var/lib/postgresql/data

volumes:
  db-data:

Gestionar Volúmenes

# Listar volúmenes
docker volume ls

# Crear volumen
docker volume create mi-volumen

# Inspeccionar volumen
docker volume inspect mi-volumen

# Eliminar volumen
docker volume rm mi-volumen

# Eliminar volúmenes no usados
docker volume prune

Ejemplo Completo: MongoDB con Persistencia

# Crear volumen
docker volume create mongo-data

# Ejecutar MongoDB con volumen
docker run -d \
  --name mongodb \
  -p 27017:27017 \
  -v mongo-data:/data/db \
  mongo:6

# Aunque elimines el contenedor, los datos permanecen en mongo-data
docker rm -f mongodb

# Crear nuevo contenedor con los mismos datos
docker run -d \
  --name mongodb-nuevo \
  -v mongo-data:/data/db \
  mongo:6
