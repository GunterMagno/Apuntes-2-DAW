4. Docker Compose

Orquestar múltiples contenedores

¿Qué es Docker Compose?

Docker Compose permite definir y ejecutar aplicaciones de múltiples contenedores usando un archivo YAML. En lugar de ejecutar comandos docker run largos, defines todo en un archivo.

Caso de uso: Aplicación web (Node.js) + Base de datos (MongoDB) + Redis (caché)

Archivo docker-compose.yml

version: '3.8'

services:
  web:
    image: nginx
    ports:
      - "8080:80"
    volumes:
      - ./html:/usr/share/nginx/html
  
  database:
    image: postgres:15
    environment:
      POSTGRES_PASSWORD: secret
    volumes:
      - db-data:/var/lib/postgresql/data

volumes:
  db-data:

Estructura

  • version: Versión del formato Compose
  • services: Contenedores a crear
  • volumes: Volúmenes compartidos
  • networks: Redes personalizadas

Comandos Docker Compose

# Levantar todos los servicios
docker-compose up

# En segundo plano
docker-compose up -d

# Reconstruir imágenes
docker-compose up --build

# Parar servicios
docker-compose down

# Ver logs
docker-compose logs

# Logs de un servicio específico
docker-compose logs web

# Ver servicios en ejecución
docker-compose ps

# Ejecutar comando en un servicio
docker-compose exec web bash

# Reiniciar servicio
docker-compose restart web

Ejemplo Completo: App Node + MongoDB

version: '3.8'

services:
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - MONGO_URL=mongodb://database:27017/myapp
    depends_on:
      - database
    volumes:
      - ./src:/app/src
    command: npm start
  
  database:
    image: mongo:6
    ports:
      - "27017:27017"
    volumes:
      - mongo-data:/data/db
    environment:
      - MONGO_INITDB_ROOT_USERNAME=admin
      - MONGO_INITDB_ROOT_PASSWORD=secret

volumes:
  mongo-data:

Redes y Dependencias

depends_on: Define orden de inicio de servicios

services:
  web:
    depends_on:
      - database  # database inicia primero

Redes personalizadas:

services:
  web:
    networks:
      - frontend
  database:
    networks:
      - backend

networks:
  frontend:
  backend:
