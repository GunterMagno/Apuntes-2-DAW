1. Introducción a Docker

Contenedorización de aplicaciones

¿Qué es Docker?

Docker es una plataforma que permite empaquetar aplicaciones junto con todas sus dependencias en contenedores. Esto garantiza que la aplicación funcione de la misma manera en cualquier entorno (desarrollo, testing, producción).

Analogía: Piensa en Docker como un contenedor de transporte marítimo. Dentro puedes meter cualquier cosa (tu aplicación, librerías, configuración), y ese contenedor se puede mover a cualquier lugar (servidor, cloud, laptop) funcionando exactamente igual.

Ventajas de Docker

  • Portabilidad: Funciona igual en cualquier sistema operativo
  • Aislamiento: Cada contenedor es independiente
  • Ligero: Consume menos recursos que máquinas virtuales
  • Rapidez: Los contenedores inician en segundos
  • Versionado: Control de versiones de imágenes

Conceptos Clave

Imagen: Plantilla de solo lectura que contiene todo lo necesario para ejecutar una aplicación.

Contenedor: Instancia ejecutable de una imagen. Es como un proceso aislado.

Dockerfile: Archivo de texto con instrucciones para construir una imagen.

Docker Hub: Repositorio público de imágenes de Docker.

Volumen: Almacenamiento persistente para los datos del contenedor.

Instalación

Windows / Mac

# Descargar Docker Desktop desde:
https://www.docker.com/products/docker-desktop

Linux (Ubuntu)

sudo apt update
sudo apt install docker.io
sudo systemctl start docker
sudo systemctl enable docker

# Verificar instalación
docker --version

Comandos Básicos

# Ver versión
docker --version

# Listar imágenes descargadas
docker images

# Descargar imagen
docker pull nginx

# Listar contenedores en ejecución
docker ps

# Listar todos los contenedores (incluso parados)
docker ps -a

# Ejecutar un contenedor
docker run nginx

# Ejecutar en segundo plano
docker run -d nginx

# Parar contenedor
docker stop <container_id>

# Eliminar contenedor
docker rm <container_id>

# Eliminar imagen
docker rmi <image_name>
