Introducción a Sass

¿Qué es Sass?

Sass (Syntactically Awesome Style Sheets) es un preprocesador CSS que extiende las capacidades de CSS con características avanzadas como variables, anidamiento, mixins, funciones y más.

Ventajas de Sass

  • ✓ Variables: Reutiliza valores (colores, fuentes, etc.)
  • ✓ Anidamiento: Código más organizado y legible
  • ✓ Mixins: Evita repetición de código
  • ✓ Funciones: Operaciones y cálculos dinámicos
  • ✓ Modularidad: Organiza tu código en múltiples archivos
  • ✓ Herencia: Comparte estilos entre selectores

Sass se compila a CSS estándar, por lo que es compatible con todos los navegadores.

Instalación

1. Usando npm (Node.js)

# Instalación global
npm install -g sass

# Instalación local en proyecto
npm install --save-dev sass

2. Verificar instalación

sass --version

3. Otras opciones

  • Standalone: Descarga desde sass-lang.com
  • VS Code: Extensión "Live Sass Compiler"
  • Build Tools: Webpack, Vite, Gulp, etc.

Sintaxis: SCSS vs Sass

SCSS (Recomendado) - Similar a CSS

// archivo.scss
$primary-color: #3498db;

.button {
    background-color: $primary-color;
    padding: 10px 20px;
    
    &:hover {
        background-color: darken($primary-color, 10%);
    }
}

Sass (Indentada) - Sin llaves ni punto y coma

// archivo.sass
$primary-color: #3498db

.button
    background-color: $primary-color
    padding: 10px 20px
    
    &:hover
        background-color: darken($primary-color, 10%)
Recomendación: Usa SCSS (.scss) ya que es más similar a CSS y más popular en la comunidad.

Variables

Las variables permiten almacenar valores que puedes reutilizar en todo tu código.

// Definir variables
$primary-color: #007bff;
$secondary-color: #6c757d;
$font-stack: 'Helvetica', Arial, sans-serif;
$base-padding: 16px;
$border-radius: 8px;

// Usar variables
.button {
    background-color: $primary-color;
    font-family: $font-stack;
    padding: $base-padding;
    border-radius: $border-radius;
}

.card {
    border: 1px solid $secondary-color;
    padding: $base-padding * 2; // Operaciones matemáticas
    border-radius: $border-radius;
}

Alcance de Variables

$global-color: blue;

.container {
    $local-color: red; // Solo disponible dentro de .container
    
    color: $global-color;
    background: $local-color;
    
    .child {
        color: $local-color; // Accesible en descendientes
    }
}

.other {
    color: $global-color; // ✓
    // color: $local-color; // ✗ Error - no definida
}

Anidamiento

Refleja la jerarquía HTML en tu CSS, haciendo el código más organizado.

// Sass
.navbar {
    background: #333;
    padding: 20px;
    
    .logo {
        font-size: 24px;
        color: white;
    }
    
    .nav-menu {
        list-style: none;
        
        li {
            display: inline-block;
            margin: 0 10px;
            
            a {
                color: white;
                text-decoration: none;
                
                &:hover {
                    color: #007bff;
                }
            }
        }
    }
}

// Compilado a CSS:
.navbar { background: #333; padding: 20px; }
.navbar .logo { font-size: 24px; color: white; }
.navbar .nav-menu { list-style: none; }
.navbar .nav-menu li { display: inline-block; margin: 0 10px; }
.navbar .nav-menu li a { color: white; text-decoration: none; }
.navbar .nav-menu li a:hover { color: #007bff; }

Operador & (selector padre)

.button {
    background: blue;
    
    &:hover { background: darkblue; }
    &:active { background: navy; }
    &.primary { background: green; }
    &-large { padding: 20px; } // .button-large
    .icon & { margin-left: 10px; } // .icon .button
}
⚠️ Evita anidar demasiado: Máximo 3-4 niveles de profundidad para mantener la especificidad bajo control.

Mixins

Los mixins permiten reutilizar grupos de declaraciones CSS con parámetros opcionales.

Mixin básico

@mixin flex-center {
    display: flex;
    justify-content: center;
    align-items: center;
}

.container {
    @include flex-center;
    height: 100vh;
}

Mixin con parámetros

@mixin button($bg-color, $text-color: white) {
    background-color: $bg-color;
    color: $text-color;
    padding: 10px 20px;
    border-radius: 5px;
    border: none;
    cursor: pointer;
    
    &:hover {
        background-color: darken($bg-color, 10%);
    }
}

.btn-primary {
    @include button(#007bff);
}

.btn-success {
    @include button(#28a745, black);
}

Mixin con contenido (@content)

@mixin responsive($breakpoint) {
    @if $breakpoint == mobile {
        @media (max-width: 768px) { @content; }
    }
    @else if $breakpoint == tablet {
        @media (max-width: 1024px) { @content; }
    }
    @else if $breakpoint == desktop {
        @media (min-width: 1025px) { @content; }
    }
}

.container {
    width: 100%;
    
    @include responsive(mobile) {
        padding: 10px;
    }
    
    @include responsive(desktop) {
        padding: 20px;
        max-width: 1200px;
    }
}

@extend y Herencia

Comparte estilos entre selectores sin duplicar código.

.message {
    padding: 15px;
    border: 1px solid #ccc;
    border-radius: 4px;
}

.success {
    @extend .message;
    border-color: #28a745;
    background-color: #d4edda;
}

.error {
    @extend .message;
    border-color: #dc3545;
    background-color: #f8d7da;
}

.warning {
    @extend .message;
    border-color: #ffc107;
    background-color: #fff3cd;
}

Placeholders (%) - Mejor práctica

%message-base {
    padding: 15px;
    border: 1px solid #ccc;
    border-radius: 4px;
}

.success {
    @extend %message-base;
    border-color: #28a745;
}

.error {
    @extend %message-base;
    border-color: #dc3545;
}

// %message-base no aparece en el CSS compilado

Módulos y @use

Organiza tu código en múltiples archivos usando el sistema moderno de módulos.

Estructura de archivos

// _variables.scss
$primary-color: #007bff;
$secondary-color: #6c757d;

// _mixins.scss
@mixin flex-center {
    display: flex;
    justify-content: center;
    align-items: center;
}

// styles.scss (archivo principal)
@use 'variables';
@use 'mixins';

.container {
    background: variables.$primary-color;
    @include mixins.flex-center;
}

Namespace personalizado

@use 'variables' as vars;
@use 'mixins' as *; // Sin namespace

.button {
    color: vars.$primary-color;
    @include flex-center; // Sin prefijo
}
@use vs @import: @use es el sistema moderno y recomendado. @import está deprecado.

Funciones

Funciones de Color

$base-color: #007bff;

.button {
    background: $base-color;
    
    &:hover {
        background: darken($base-color, 10%);
    }
    
    &:active {
        background: lighten($base-color, 10%);
    }
}

.text {
    color: mix(red, blue, 50%); // Mezcla colores
    background: rgba($base-color, 0.5); // Añade transparencia
}

Funciones Matemáticas

$base-size: 16px;

.text-small { font-size: $base-size * 0.875; }
.text-large { font-size: $base-size * 1.25; }

// Funciones incorporadas
.box {
    width: percentage(0.5); // 50%
    padding: round(15.6px); // 16px
    margin: ceil(10.2px); // 11px
}

Funciones Personalizadas

@function rem($pixels) {
    @return $pixels / 16px * 1rem;
}

.heading {
    font-size: rem(24px); // 1.5rem
    margin-bottom: rem(16px); // 1rem
}

Compilación

Comandos básicos

# Compilar archivo único
sass input.scss output.css

# Watch mode (compilación automática)
sass --watch input.scss:output.css

# Watch carpeta completa
sass --watch sass:css

# Estilo de salida
sass --style=compressed input.scss output.css

Estilos de salida

  • expanded: CSS legible (default)
  • compressed: CSS minificado para producción

Configuración en package.json

{
  "scripts": {
    "sass": "sass --watch sass:css",
    "sass:build": "sass --style=compressed sass:css"
  },
  "devDependencies": {
    "sass": "^1.69.0"
  }
}

Ejecutar

npm run sass        # Desarrollo
npm run sass:build  # Producción
