@scope - Estilos con Alcance Limitado

@scope es una nueva funcionalidad de CSS que permite limitar el alcance de los estilos a una región específica del DOM, evitando conflictos de estilos y mejorando la modularidad del código.

¿Qué es @scope?

La regla @scope permite definir estilos que solo se aplican dentro de un contexto específico del documento HTML. Esto es especialmente útil en aplicaciones grandes donde diferentes componentes pueden tener clases con nombres similares.

Dato importante: @scope es parte de la especificación CSS Cascading and Inheritance Level 6 y proporciona una alternativa nativa a metodologías como BEM o CSS Modules.

Problema que resuelve

Sin @scope, los estilos globales pueden afectar elementos no deseados:

/* Sin @scope - Afecta a TODOS los .title */
.title {
    color: blue;
    font-size: 24px;
}

/* Esto afectará a .title en toda la página */

Sintaxis Básica

@scope (scope-root) {
    /* Estilos que solo se aplican dentro de scope-root */
    .elemento {
        propiedad: valor;
    }
}

/* Con límite de alcance */
@scope (scope-root) to (scope-limit) {
    /* Estilos aplicados entre scope-root y scope-limit */
    .elemento {
        propiedad: valor;
    }
}

Componentes principales:

  • scope-root: Define dónde comienza el alcance (el contenedor)
  • scope-limit (to): Define dónde termina el alcance (opcional)
  • Selectores internos: Solo afectan elementos dentro del scope

¿Cómo Funciona?

@scope crea un "contexto de estilo" donde los selectores solo coinciden con elementos que son descendientes del scope root (y opcionalmente, no descendientes del scope limit).

Ejemplo Visual

/* HTML */
<div class="card">
    <h2 class="title">Título de la Card</h2>
    <p>Contenido de la card</p>
</div>

<div class="sidebar">
    <h2 class="title">Título del Sidebar</h2>
    <p>Contenido del sidebar</p>
</div>

/* CSS con @scope */
@scope (.card) {
    .title {
        color: blue;
        font-size: 24px;
    }
}

@scope (.sidebar) {
    .title {
        color: red;
        font-size: 18px;
    }
}

/* Resultado:
   - .title dentro de .card será azul y 24px
   - .title dentro de .sidebar será rojo y 18px
   - NO hay conflictos entre ambos */

Scope Root - El Contenedor

El scope root define el elemento contenedor donde comenzarán a aplicarse los estilos.

/* Scope root con selector de clase */
@scope (.component) {
    h1 { color: navy; }
    p { line-height: 1.6; }
}

/* Scope root con selector de ID */
@scope (#main-content) {
    .button { background: green; }
}

/* Scope root con selector de atributo */
@scope ([data-theme="dark"]) {
    .text { color: white; }
}

/* Scope root con selector complejo */
@scope (article.blog-post) {
    img { border-radius: 8px; }
}

Selector :scope

Dentro de un @scope, puedes usar el selector :scope para referirte al elemento root:

@scope (.card) {
    /* Aplica estilos al propio elemento .card */
    :scope {
        border: 1px solid #ccc;
        padding: 20px;
    }
    
    /* Aplica a descendientes */
    .title {
        font-weight: bold;
    }
}

Scope Limit - El Límite

El scope limit (usando to) define un límite donde los estilos dejan de aplicarse, incluso dentro del scope root.

/* HTML */
<article class="post">
    <h1>Título del Post</h1>
    <p>Este párrafo estará estilizado</p>
    
    <div class="comments">
        <h2>Comentarios</h2>
        <p>Este párrafo NO estará estilizado</p>
    </div>
</article>

/* CSS con scope limit */
@scope (.post) to (.comments) {
    p {
        color: blue;
        font-size: 16px;
    }
}

/* Resultado:
   - El <p> dentro de .post pero FUERA de .comments: azul
   - El <p> dentro de .comments: estilo por defecto (no afectado) */

Casos de uso del scope limit

  • Excluir secciones anidadas (ej: comentarios, widgets embebidos)
  • Prevenir que estilos afecten componentes internos independientes
  • Crear "zonas neutras" dentro de un scope más grande

Ejemplo práctico: Card con sección especial

@scope (.product-card) to (.reviews) {
    /* Estilos para el contenido de la card */
    h3 { color: #333; }
    p { margin: 10px 0; }
}

@scope (.reviews) {
    /* Estilos específicos para las reviews */
    h3 { color: #0066cc; }
    p { font-style: italic; }
}

Especificidad y Cascade

Los estilos dentro de @scope tienen un comportamiento especial respecto a la especificidad:

Regla importante: Los estilos dentro de @scope tienen mayor "proximidad" al elemento, lo que les da prioridad sobre estilos globales con la misma especificidad.
Tipo de Estilo Especificidad Prioridad
Estilo global 0-1-0 (.class) Baja
Estilo dentro de @scope 0-1-0 (.class) Media (mayor proximidad)
Estilo con mayor especificidad 0-2-0 (.class.class) Alta
Estilo inline 1-0-0 Muy alta

Ejemplo de Especificidad

/* Estilo global */
.button {
    background: gray; /* No se aplicará */
}

/* Estilo dentro de scope - GANA por proximidad */
@scope (.toolbar) {
    .button {
        background: blue; /* Este se aplica */
    }
}

/* Pero un selector más específico aún gana */
.toolbar .button.primary {
    background: green; /* Este gana sobre el scope */
}

Scopes Anidados

Cuando tienes scopes anidados, el scope más cercano tiene prioridad:

@scope (.outer) {
    .title {
        color: blue;
    }
    
    @scope (.inner) {
        .title {
            color: red; /* Gana por estar más cerca */
        }
    }
}

Casos de Uso Comunes

1. Sistema de Componentes

Crear componentes reutilizables sin conflictos de nombres:

@scope (.modal) {
    .header { /* Solo afecta headers dentro de modales */ }
    .footer { /* Solo afecta footers dentro de modales */ }
}

2. Temas Contextuales

Aplicar diferentes temas a secciones específicas:

@scope (.admin-panel) {
    :scope { --primary-color: #8b0000; }
}

@scope (.user-dashboard) {
    :scope { --primary-color: #0066cc; }
}

3. Protección de Estilos

Proteger componentes de estilos globales no deseados:

@scope (.widget) to (.widget-excluded) {
    /* Estilos solo dentro de .widget pero no en .widget-excluded */
    * {
        box-sizing: border-box;
    }
}

4. Layouts Complejos

Diferentes estilos para diferentes regiones de la página:

@scope (header) {
    nav { /* Navegación del header */ }
}

@scope (footer) {
    nav { /* Navegación del footer, estilos diferentes */ }
}

5. Microfrontends

Aislar estilos de diferentes aplicaciones en la misma página:

@scope ([data-app="app1"]) {
    /* Estilos exclusivos para app1 */
}

@scope ([data-app="app2"]) {
    /* Estilos exclusivos para app2 */
}

Ventajas de Usar @scope

  • Encapsulación: Los estilos están aislados y no afectan el resto de la página
  • Modularidad: Facilita la creación de componentes independientes y reutilizables
  • Código más limpio: No necesitas prefijos largos o metodologías como BEM
  • Rendimiento: El navegador puede optimizar mejor los estilos al conocer su alcance
  • Mantenibilidad: Más fácil entender y modificar estilos específicos de componentes
  • Trabajo en equipo: Reduce conflictos cuando varios desarrolladores trabajan en paralelo
  • Nativo: No requiere preprocesadores ni herramientas adicionales

Comparación con otras soluciones

Solución Ventajas Desventajas
@scope (nativo) No requiere herramientas, rendimiento optimizado Soporte limitado en navegadores antiguos
BEM (metodología) Compatible con todos los navegadores Nombres de clases largos, verboso
CSS Modules Buen soporte, automatizado Requiere build step y configuración
Shadow DOM Encapsulación total Complejidad, dificulta estilos globales

Compatibilidad con Navegadores

Advertencia: @scope es una funcionalidad relativamente nueva. Verifica la compatibilidad actual antes de usar en producción.

Estado actual (Noviembre 2025)

  • Chrome/Edge: Soporte desde versión 118+ (Septiembre 2023)
  • Safari: Soporte desde versión 17.4+ (Marzo 2024)
  • Firefox: En desarrollo, disponible con flag experimental
  • Opera: Soportado (basado en Chromium)

Detección de soporte

/* JavaScript */
if (CSS.supports('selector(:scope)')) {
    console.log('@scope es soportado');
} else {
    console.log('@scope NO es soportado');
}

/* CSS con fallback */
.component .title {
    color: blue; /* Fallback */
}

@supports (selector(:scope)) {
    @scope (.component) {
        .title {
            color: navy; /* Usado si @scope está soportado */
        }
    }
}

Estrategias de Fallback

/* 1. Selectores tradicionales como fallback */
.card .title { color: black; }

@scope (.card) {
    .title { color: blue; }
}

/* 2. Progressive enhancement */
.button {
    /* Estilos base para todos */
    padding: 10px 20px;
}

@scope (.premium-section) {
    /* Mejoras solo si el navegador lo soporta */
    .button {
        background: linear-gradient(45deg, gold, orange);
    }
}
Recomendación: Para proyectos en producción, considera usar PostCSS con un plugin que transforme @scope a selectores tradicionales para navegadores sin soporte.
