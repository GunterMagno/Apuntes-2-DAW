Resumen de CSS & Sass

CSS - Cascading Style Sheets

Conceptos Fundamentales

  • CSS controla la presentación visual de HTML
  • Separación de contenido (HTML) y presentación (CSS)
  • Sistema de cascada y especificidad

Selectores Principales

  • Elemento: p { }
  • Clase: .nombre { }
  • ID: #nombre { }
  • Atributo: [type="text"] { }
  • Pseudo-clases: :hover, :focus, :nth-child()

Box Model

  • Content: Contenido del elemento
  • Padding: Espacio interno
  • Border: Borde del elemento
  • Margin: Espacio externo
  • box-sizing: border-box; - Recomendado

@scope - Nuevo en CSS

  • Limita el alcance de estilos a regiones específicas del DOM
  • @scope (.contenedor) { ... }
  • @scope (.contenedor) to (.limite) { ... }
  • Mejora la modularidad y evita conflictos de estilos
  • Soporte: Chrome 118+, Safari 17.4+

Sass - Preprocesador CSS

¿Qué es Sass?

  • Preprocesador CSS que extiende las capacidades de CSS estándar
  • Dos sintaxis: SCSS (similar a CSS) y Sass (indentada)
  • Se compila a CSS estándar para los navegadores

Variables

  • $primary-color: #007bff;
  • $font-stack: 'Helvetica', sans-serif;
  • Reutilización de valores en todo el proyecto
  • Operaciones matemáticas: $base-padding * 2

Anidamiento

  • Refleja la jerarquía HTML en CSS
  • Usa & para referenciar el selector padre
  • Evita el anidamiento excesivo (máximo 3-4 niveles)

Mixins

  • @mixin nombre($param) { ... }
  • @include nombre(valor);
  • Reutiliza bloques de código con parámetros
  • Ideal para vendor prefixes y responsive design

Funciones

  • Color: darken(), lighten(), mix()
  • Matemáticas: percentage(), round()
  • Personalizadas: @function nombre() { @return ... }

@extend y Herencia

  • @extend .clase; - Comparte estilos
  • Placeholders: %nombre { ... }
  • Reduce la duplicación de código

Módulos (@use)

  • @use 'archivo'; - Sistema moderno de importación
  • @use 'archivo' as nombre; - Alias
  • archivo.$variable - Acceso a variables
  • Reemplaza a @import (deprecado)

Compilación

  • CLI: sass input.scss output.css
  • Watch: sass --watch input.scss:output.css
  • Estilos: expanded (legible), compressed (producción)

Mejores Prácticas

  • Usa SCSS en lugar de sintaxis indentada (más compatible con CSS)
  • Organiza tu código en múltiples archivos con @use
  • Define variables para valores reutilizables
  • Crea mixins para patrones repetitivos
  • Evita anidar más de 3-4 niveles
  • Usa placeholders (%) en lugar de clases para @extend
  • Aprovecha @scope en CSS para componentes aislados
  • Compila en modo compressed para producción
