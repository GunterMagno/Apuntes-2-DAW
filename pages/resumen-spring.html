Resumen Rápido: Spring Boot

Referencia rápida de anotaciones, comandos Maven y conceptos clave

Comandos Maven más utilizados

Maven es la herramienta de gestión de dependencias y construcción más usada con Spring Boot. Estos son los comandos esenciales.

Comando Descripción Cuándo usarlo
mvn clean Limpia archivos compilados anteriores (carpeta target/) Antes de compilar de nuevo para asegurar un build limpio
mvn compile Compila el código fuente del proyecto Para verificar que no hay errores de compilación
mvn test Ejecuta todos los tests unitarios Verificar que los cambios no rompieron nada
mvn package Compila, testea y crea el JAR/WAR Crear el archivo ejecutable de tu aplicación
mvn spring-boot:run Ejecuta la aplicación Spring Boot directamente Durante desarrollo para probar cambios rápidamente
mvn clean install Limpia, compila, testea e instala en repositorio local El comando más completo para builds definitivos
mvn dependency:tree Muestra el árbol de dependencias del proyecto Depurar conflictos de versiones de librerías
java -jar target/app.jar Ejecuta el JAR generado por Maven Ejecutar la aplicación en producción

Anotaciones principales de Spring Boot

Las anotaciones son fundamentales en Spring Boot. Configuran el comportamiento sin necesidad de XML.

Anotación Dónde se usa Qué hace
@SpringBootApplication Clase principal Combina @Configuration, @EnableAutoConfiguration y @ComponentScan. Marca el punto de entrada de la aplicación
@RestController Clase controller Indica que es un controlador REST. Combina @Controller + @ResponseBody
@GetMapping("/ruta") Método controller Mapea peticiones HTTP GET a este método
@PostMapping("/ruta") Método controller Mapea peticiones HTTP POST (crear recursos)
@PutMapping("/ruta") Método controller Mapea peticiones HTTP PUT (actualizar recursos)
@DeleteMapping("/ruta") Método controller Mapea peticiones HTTP DELETE (eliminar recursos)
@RequestBody Parámetro método Convierte el JSON del body de la petición a un objeto Java
@PathVariable Parámetro método Extrae valores de la URL (ej: /users/{id})
@RequestParam Parámetro método Extrae parámetros de query (ej: /users?name=Juan)
@Service Clase service Marca una clase como servicio (lógica de negocio)
@Repository Interface repository Marca una clase como repositorio (acceso a datos)
@Entity Clase modelo Indica que es una entidad JPA mapeada a una tabla BD
@Id Campo entidad Marca el campo como clave primaria
@GeneratedValue Campo entidad Genera automáticamente el valor del ID
@Autowired Campo/Constructor Inyección de dependencias automática
@Configuration Clase config Indica que la clase contiene configuración de beans
@Bean Método config Define un bean gestionado por Spring

Estructura típica de proyecto

Carpeta/Archivo Contenido Propósito
src/main/java Código fuente Java Todo tu código de aplicación
controller/ Clases @RestController Manejan peticiones HTTP y devuelven respuestas
service/ Clases @Service Lógica de negocio de la aplicación
repository/ Interfaces @Repository Acceso a base de datos (extienden JpaRepository)
model/ o entity/ Clases @Entity Entidades que mapean a tablas de BD
dto/ Clases DTO (Data Transfer Object) Objetos para transferir datos entre capas
config/ Clases @Configuration Configuración de beans, seguridad, CORS, etc.
src/main/resources Recursos no-Java Archivos de configuración y estáticos
application.properties Configuración de Spring Boot Puerto, BD, logging, perfiles, etc.
static/ Archivos estáticos CSS, JS, imágenes servidos directamente
templates/ Plantillas HTML Si usas Thymeleaf para renderizar HTML
src/test/java Tests unitarios Tests con JUnit y Mockito
pom.xml Configuración Maven Dependencias, plugins, versiones

Configuraciones comunes (application.properties)

Propiedad Ejemplo de valor Qué configura
server.port 8080 Puerto donde se ejecuta la aplicación
spring.application.name mi-app Nombre de la aplicación
spring.datasource.url jdbc:h2:mem:testdb URL de conexión a la base de datos
spring.datasource.username sa Usuario de la base de datos
spring.datasource.password password Contraseña de la base de datos
spring.jpa.hibernate.ddl-auto update Cómo manejar el esquema BD (create, update, validate, none)
spring.jpa.show-sql true Mostrar las queries SQL en la consola
spring.h2.console.enabled true Habilitar consola web de H2 para ver la BD
logging.level.root INFO Nivel de logging global (TRACE, DEBUG, INFO, WARN, ERROR)
spring.profiles.active dev Perfil activo (dev, test, prod)

Códigos de respuesta HTTP

Spring Boot usa estos códigos para comunicar el resultado de las peticiones.

Código Nombre Cuándo usarlo En Spring Boot
200 OK Petición exitosa (GET, PUT) ResponseEntity.ok()
201 Created Recurso creado exitosamente (POST) ResponseEntity.status(201).body()
204 No Content Éxito sin contenido (DELETE) ResponseEntity.noContent().build()
400 Bad Request Datos inválidos en la petición ResponseEntity.badRequest()
404 Not Found Recurso no encontrado ResponseEntity.notFound().build()
500 Internal Server Error Error en el servidor Spring lo devuelve automáticamente en excepciones

Métodos principales de JpaRepository

Cuando tu repositorio extiende JpaRepository, heredas automáticamente estos métodos.

Método Qué hace Ejemplo de uso
save(entity) Guarda o actualiza una entidad repository.save(usuario)
findAll() Obtiene todas las entidades List<User> users = repository.findAll()
findById(id) Busca por ID, devuelve Optional Optional<User> user = repository.findById(1L)
existsById(id) Verifica si existe una entidad con ese ID boolean exists = repository.existsById(1L)
count() Cuenta cuántas entidades hay long total = repository.count()
deleteById(id) Elimina por ID repository.deleteById(1L)
delete(entity) Elimina una entidad repository.delete(usuario)
deleteAll() Elimina todas las entidades repository.deleteAll()

Anotaciones de Lombok

Lombok reduce código boilerplate generando automáticamente getters, setters, constructores, etc.

Anotación Qué genera Cuándo usarla
@Data @Getter, @Setter, @ToString, @EqualsAndHashCode, @RequiredArgsConstructor Para DTOs y entidades simples (la más usada)
@Getter Métodos getter para todos los campos Cuando solo necesitas getters
@Setter Métodos setter para todos los campos Cuando solo necesitas setters
@NoArgsConstructor Constructor sin parámetros Requerido por JPA para entidades
@AllArgsConstructor Constructor con todos los parámetros Para crear objetos con todos los valores
@RequiredArgsConstructor Constructor con campos final y @NonNull Para inyección de dependencias
@Builder Patrón Builder para crear objetos Crear objetos de forma fluida y legible
@ToString Método toString() Para debugging y logging
@Slf4j Logger automático (log variable) Para logging en la clase

Consejo importante

Usa este resumen como referencia rápida, pero siempre revisa la página de detalles completa para ver ejemplos de código reales y explicaciones más profundas de cada concepto.

Spring Boot tiene una curva de aprendizaje, pero una vez dominas las anotaciones y la arquitectura en capas, es muy productivo. Practica creando APIs REST simples y ve añadiendo complejidad gradualmente.
