Hooks principales

Los Hooks permiten usar estado y otras características de React en componentes funcionales.

Hook Qué hace Cuándo usarlo
useState() Añade estado al componente Para datos que cambian (formularios, toggles, contadores)
useEffect() Ejecuta efectos secundarios Llamadas API, suscripciones, manipulación DOM
useContext() Consume un Context Acceder a datos globales (tema, usuario, idioma)
useRef() Crea referencia mutable Acceder al DOM directamente, valores que no causan re-render
useMemo() Memoriza valor calculado Optimizar cálculos costosos
useCallback() Memoriza función Evitar recrear funciones en cada render
useReducer() Alternativa a useState para lógica compleja Estado con múltiples sub-valores o lógica compleja

useState - Manejo de estado

Sintaxis Descripción Ejemplo
const [state, setState] Declarar variable de estado const [count, setCount] = useState(0)
setState(newValue) Actualizar estado con valor setCount(5)
setState(prev => ...) Actualizar basado en valor anterior setCount(prev => prev + 1)
useState(initialValue) Valor inicial del estado useState(0), useState(''), useState([])

useEffect - Efectos secundarios

Patrón Cuándo se ejecuta Ejemplo
useEffect(() => {}) Después de cada render useEffect(() => { console.log('Render') })
useEffect(() => {}, []) Solo una vez (al montar) useEffect(() => { fetchData() }, [])
useEffect(() => {}, [dep]) Cuando cambia dependencia useEffect(() => { ... }, [userId])
return () => {} Cleanup (al desmontar) return () => { clearInterval(id) }

JSX - Sintaxis y reglas

Concepto Regla/Sintaxis Ejemplo
Expresiones Usar {} para JavaScript <h1>{title}</h1>, <p>{2 + 2}</p>
className Usar className en vez de class <div className="container">
htmlFor Usar htmlFor en vez de for <label htmlFor="name">
Estilos inline Objeto JavaScript con camelCase <div style={{backgroundColor: 'blue'}}>

Fragmentos Agrupar sin div extra <>...</> o <React.Fragment>
Comentarios Entre llaves y /* */ {/* Este es un comentario */}

Props - Pasar datos a componentes

Concepto Descripción Ejemplo
Pasar props Como atributos del componente <User name="Juan" age={25} />
Recibir props Parámetro del componente function User(props) { ... }
Destructuring Extraer props directamente function User({ name, age }) { ... }
props.children Contenido entre tags <Card>{props.children}</Card>
Default props Valores por defecto function User({ name = "Anónimo" }) { ... }

Evento Sintaxis Ejemplo

Ciclo de vida con useEffect

Fase Equivalente con useEffect
componentDidMount useEffect(() => { ... }, [])
componentDidUpdate useEffect(() => { ... }, [dependency])
componentWillUnmount useEffect(() => { return () => { cleanup } }, [])

Consejo importante

React se basa en componentes: divide tu UI en piezas pequeñas y reutilizables. Cada componente debe tener una responsabilidad clara.

El flujo de datos es unidireccional: los datos van de padres a hijos (props). Los hijos comunican al padre mediante callbacks.

No mutes el estado directamente: siempre usa setState. Para objetos y arrays, crea copias nuevas con spread operator.

useEffect puede ser complejo: asegúrate de entender las dependencias y el cleanup. Es el source más común de bugs.
