Resumen Rápido: Java

Referencia rápida de comandos, sintaxis y conceptos clave

Comandos más utilizados

Esta tabla contiene los comandos de Java que usarás con más frecuencia en tu día a día como desarrollador.

Comando Descripción Ejemplo práctico
javac MiClase.java Compila un archivo .java a bytecode (.class) javac Calculadora.java genera Calculadora.class
java MiClase Ejecuta una clase compilada (debe tener método main) java Calculadora ejecuta el programa
java -version Muestra la versión de Java instalada en tu sistema Útil para verificar que Java está instalado correctamente
javac -d bin src/*.java Compila todos los archivos .java de src/ y guarda los .class en bin/ Organiza mejor los archivos compilados
jar -cvf app.jar *.class Crea un archivo JAR con todas las clases compiladas Empaqueta tu aplicación para distribuir
java -jar app.jar Ejecuta un archivo JAR ejecutable El JAR debe tener un MANIFEST con la clase Main
javap -c MiClase Desensambla y muestra el bytecode de una clase Útil para entender cómo funciona internamente
jshell Abre el shell interactivo de Java (desde Java 9+) Perfecto para probar código rápidamente

Tipos de datos primitivos

Java tiene 8 tipos primitivos básicos. Es importante conocer su rango y cuándo usar cada uno para optimizar memoria.

Tipo Tamaño Rango Cuándo usarlo
byte 8 bits -128 a 127 Para ahorrar memoria con números muy pequeños
short 16 bits -32,768 a 32,767 Números pequeños cuando byte no es suficiente
int 32 bits -2,147,483,648 a 2,147,483,647 Más usado para números enteros normales
long 64 bits -9,223,372,036,854,775,808 a 9,223,372,036,854,775,807 Números muy grandes (usar sufijo L: 123456789L)
float 32 bits ~7 dígitos decimales Decimales con precisión limitada (usar sufijo f: 3.14f)
double 64 bits ~15 dígitos decimales Más usado para números decimales
char 16 bits Unicode 0 a 65,535 Un solo carácter: 'A', '5', '@'
boolean 1 bit true o false Valores lógicos, condicionales, banderas

Estructuras de control

Las estructuras que controlan el flujo de ejecución del programa.

Estructura Sintaxis básica Cuándo usarla
if if (condicion) { ... } Ejecutar código solo si se cumple una condición
if-else if (condicion) { ... } else { ... } Dos caminos posibles según la condición
else if if (...) { } else if (...) { } else { } Múltiples condiciones secuenciales
switch switch (variable) { case valor: ... break; } Múltiples opciones basadas en un valor específico
for for (int i = 0; i < 10; i++) { ... } Repetir un número conocido de veces
while while (condicion) { ... } Repetir mientras se cumpla una condición
do-while do { ... } while (condicion); Ejecutar al menos una vez y repetir si se cumple
for-each for (Tipo elem : coleccion) { ... } Recorrer todos los elementos de una colección

Colecciones más usadas

Las estructuras de datos que usarás constantemente para almacenar múltiples elementos.

Colección Características Cuándo usarla Métodos clave
ArrayList<T> Lista dinámica, permite duplicados, mantiene orden Cuando necesitas una lista que puede crecer add(), get(), remove(), size(), contains()
LinkedList<T> Lista enlazada, mejor para inserción/eliminación Muchas operaciones de insertar/eliminar en medio addFirst(), addLast(), removeFirst(), removeLast()
HashSet<T> No permite duplicados, sin orden garantizado Cuando necesitas elementos únicos sin importar el orden add(), remove(), contains(), size()
HashMap<K,V> Pares clave-valor, claves únicas Buscar valores por una clave específica put(), get(), remove(), containsKey(), keySet()
TreeSet<T> Sin duplicados, elementos ordenados Elementos únicos que necesitan estar ordenados add(), remove(), first(), last()

Conceptos de POO (Programación Orientada a Objetos)

Concepto Qué es Para qué sirve Palabra clave
Clase Plantilla o molde para crear objetos Define la estructura y comportamiento de los objetos class MiClase { }
Objeto Instancia concreta de una clase Representa una entidad específica con datos propios new MiClase()
Encapsulación Ocultar datos internos, exponer solo lo necesario Proteger datos y controlar el acceso private, public, protected
Herencia Una clase puede heredar de otra (padre-hijo) Reutilizar código y crear jerarquías extends
Polimorfismo Un método puede tener diferentes implementaciones Mismo método se comporta diferente según el objeto @Override
Abstracción Definir estructura sin implementación completa Crear contratos que otras clases deben cumplir abstract, interface

Modificador Misma clase Mismo paquete Subclases Todos Cuándo usarlo

Manejo de excepciones

Cómo manejar errores de forma controlada en tus programas.

Bloque Propósito Cuándo se ejecuta
try { } Código que puede lanzar una excepción Siempre se intenta ejecutar
catch (Exception e) { } Captura y maneja la excepción Solo si ocurre un error en el try
finally { } Código que siempre se ejecuta (limpieza) Siempre, haya o no excepción
throw new Exception() Lanza manualmente una excepción Cuando detectas una condición de error
throws Exception Declara que un método puede lanzar excepciones En la firma del método

Métodos comunes de String

Los métodos de String que usarás todo el tiempo para manipular texto.

Método Qué hace Ejemplo
length() Devuelve el número de caracteres "Hola".length() → 4
charAt(i) Devuelve el carácter en la posición i "Hola".charAt(0) → 'H'
substring(inicio, fin) Extrae una porción del texto "Hola".substring(0, 2) → "Ho"
toLowerCase() Convierte a minúsculas "HOLA".toLowerCase() → "hola"
toUpperCase() Convierte a mayúsculas "hola".toUpperCase() → "HOLA"
trim() Elimina espacios al inicio y final " hola ".trim() → "hola"
replace(old, new) Reemplaza texto "hola".replace('o', 'a') → "hala"
split(separador) Divide el string en un array "a,b,c".split(",") → ["a", "b", "c"]
contains(texto) Verifica si contiene un texto "hola".contains("ol") → true
equals(otro) Compara contenido (¡NO uses ==!) "hola".equals("hola") → true
