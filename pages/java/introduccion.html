1. Introducción a Java

Conoce los fundamentos y la arquitectura de Java

¿Qué es Java?

Java es un lenguaje de programación de propósito general y orientado a objetos creado por Sun Microsystems en 1995 (ahora propiedad de Oracle Corporation). Fue diseñado con una filosofía clara: "Write Once, Run Anywhere" (Escribe una vez, ejecuta en cualquier lugar).

Esto significa que cuando escribes código Java y lo compilas, el resultado puede ejecutarse en cualquier sistema operativo (Windows, Linux, macOS, etc.) sin necesidad de modificar el código. Esta portabilidad es una de las razones por las que Java es tan popular en el desarrollo empresarial y de aplicaciones móviles.

¿Por qué aprender Java?

  • Ampliamente utilizado: Java es uno de los lenguajes más usados en el mundo, especialmente en aplicaciones empresariales
  • Desarrollo Android: El lenguaje principal para desarrollar aplicaciones Android (junto con Kotlin)
  • Gran demanda laboral: Muchas empresas buscan desarrolladores Java
  • Ecosistema robusto: Miles de bibliotecas y frameworks disponibles (Spring, Hibernate, etc.)
  • Comunidad activa: Gran comunidad que ofrece soporte, tutoriales y soluciones

Características Principales

1. Orientado a Objetos

En Java, todo gira alrededor de objetos. Un objeto es una entidad que tiene estado (atributos o propiedades) y comportamiento (métodos o funciones). Por ejemplo, si modelamos un "Coche" como objeto:

  • Estado: color, marca, velocidad actual
  • Comportamiento: acelerar(), frenar(), cambiarMarcha()

Esta forma de programar nos permite organizar el código de manera más intuitiva, modelando conceptos del mundo real.

2. Multiplataforma

Gracias a la JVM (Java Virtual Machine), el código Java puede ejecutarse en cualquier dispositivo que tenga una JVM instalada. El proceso es:

  1. Escribes el código fuente en archivos .java
  2. El compilador lo convierte a bytecode (archivos .class)
  3. La JVM interpreta ese bytecode y lo ejecuta en tu sistema operativo

El bytecode es independiente de la plataforma, por eso puedes compilar en Windows y ejecutar en Linux sin cambios.

3. Seguro y Robusto

Java incorpora múltiples características de seguridad:

  • Gestión automática de memoria: El Garbage Collector libera memoria automáticamente, evitando fugas de memoria
  • Sin punteros explícitos: A diferencia de C/C++, no puedes manipular memoria directamente, lo que previene muchos errores
  • Verificación de bytecode: Antes de ejecutar, la JVM verifica que el código sea seguro
  • Manejo de excepciones: Sistema robusto para gestionar errores sin que la aplicación colapse

4. Multihilo (Multithreading)

Java tiene soporte nativo para programación concurrente, lo que permite que tu aplicación realice múltiples tareas simultáneamente. Por ejemplo:

  • Una aplicación puede estar descargando un archivo mientras actualiza la interfaz de usuario
  • Un servidor puede atender a múltiples clientes al mismo tiempo

Esto se logra mediante hilos (threads), que son flujos de ejecución independientes dentro del mismo programa.

¿Cómo Funciona Java?

Para entender Java, es importante conocer su arquitectura de dos fases: compilación y ejecución.

Fase 1: Compilación

Cuando escribes código Java, lo haces en archivos de texto plano con extensión .java. Este código lo entienden los humanos, pero no las máquinas. Aquí entra el compilador (javac).

¿Qué hace el compilador?

  • Lee tu archivo .java
  • Verifica que la sintaxis sea correcta
  • Traduce el código a bytecode
  • Genera un archivo .class con el bytecode

El bytecode es un lenguaje intermedio que no es código máquina nativo, pero tampoco es código fuente legible. Es el lenguaje que entiende la JVM. 

// Ejemplo: Archivo HolaMundo.java
public class HolaMundo {
    public static void main(String[] args) {
        System.out.println("¡Hola Mundo!");
    }
}

Para compilar este archivo, usas el comando:

javac HolaMundo.java

Esto generará un archivo HolaMundo.class que contiene el bytecode.

La JVM - Java Virtual Machine

La JVM es el corazón de Java. Es un programa que actúa como una "computadora virtual" capaz de ejecutar bytecode. Cada sistema operativo tiene su propia JVM (JVM para Windows, JVM para Linux, etc.), pero todas entienden el mismo bytecode.

Componentes principales de la JVM:

  • Class Loader: Carga las clases (.class) en memoria cuando son necesarias. No carga todas las clases al inicio, solo las que se van usando.
  • Bytecode Verifier: Verifica que el bytecode sea válido y seguro antes de ejecutarlo. Comprueba que no haya operaciones ilegales como acceso a memoria prohibido.
  • JIT Compiler (Just-In-Time): Optimiza el rendimiento. Convierte las partes del bytecode que se ejecutan frecuentemente a código máquina nativo, haciendo que el programa sea más rápido.
  • Garbage Collector: Se encarga de la gestión automática de memoria. Busca objetos que ya no se usan y libera la memoria que ocupan, evitando fugas de memoria.

Fase 2: Ejecución

Una vez compilado, puedes ejecutar tu programa con el comando java: 

java HolaMundo

Nota: No incluyes la extensión .class en el comando, solo el nombre de la clase.

Al ejecutar este comando, la JVM:

  1. Carga la clase HolaMundo.class
  2. Busca el método main() (punto de entrada del programa)
  3. Empieza a ejecutar las instrucciones del método main
  4. En este caso, imprime "¡Hola Mundo!" en la consola

Mi Primer Programa en Java

Vamos a analizar en detalle el programa "Hola Mundo" para entender cada componente: 

public class HolaMundo {
    public static void main(String[] args) {
        System.out.println("¡Hola Mundo!");
    }
}

Desglose línea por línea:

1. public class HolaMundo {

  • public: Es un modificador de acceso. Significa que esta clase es accesible desde cualquier lugar.
  • class: Palabra clave que indica que estamos definiendo una clase.
  • HolaMundo: El nombre de la clase. IMPORTANTE: Debe coincidir exactamente con el nombre del archivo (HolaMundo.java).
  • {: Abre el bloque de código de la clase.

2. public static void main(String[] args) {

Esta es la firma del método main, el punto de entrada de cualquier aplicación Java. Es obligatorio y debe escribirse exactamente así:

  • public: El método debe ser público para que la JVM pueda acceder a él.
  • static: Indica que este método pertenece a la clase, no a instancias de la clase. La JVM puede llamarlo sin crear un objeto.
  • void: Indica que el método no devuelve ningún valor.
  • main: El nombre del método. La JVM busca específicamente este nombre.
  • String[] args: Un parámetro que es un array de Strings. Permite pasar argumentos al programa desde la línea de comandos.

3. System.out.println("¡Hola Mundo!");

  • System: Una clase del paquete java.lang que proporciona acceso al sistema.
  • out: Un objeto de tipo PrintStream que representa la salida estándar (consola).
  • println: Un método que imprime texto en la consola y añade un salto de línea al final.
  • "¡Hola Mundo!": El String (cadena de texto) que queremos imprimir.
  • ;: Todas las instrucciones en Java terminan con punto y coma.

4. Las llaves de cierre }

La primera cierra el método main, la segunda cierra la clase HolaMundo.

Errores Comunes

  • Nombre del archivo ≠ nombre de la clase: Causará un error de compilación
  • Olvidar el punto y coma (;) al final de las instrucciones
  • Mayúsculas/minúsculas: Java es case-sensitive. Main ≠ main
  • Olvidar las llaves de cierre: Cada { debe tener su }

Estructura Básica de un Programa Java

Todo programa Java sigue una estructura organizada. Aquí te muestro los elementos clave: 

// 1. Declaración del paquete (opcional)
package com.miempresa.proyecto;

// 2. Importaciones (opcional)
import java.util.ArrayList;
import java.util.Scanner;

// 3. Declaración de la clase (obligatorio)
public class MiPrograma {
    
    // 4. Atributos de la clase
    private String nombre;
    private int edad;
    
    // 5. Constructor
    public MiPrograma(String nombre, int edad) {
        this.nombre = nombre;
        this.edad = edad;
    }
    
    // 6. Métodos
    public void saludar() {
        System.out.println("Hola, soy " + nombre);
    }
    
    // 7. Método main (punto de entrada)
    public static void main(String[] args) {
        MiPrograma persona = new MiPrograma("Ana", 25);
        persona.saludar();
    }
}

Explicación de cada sección:

1. Paquete (package)

Los paquetes son como carpetas que organizan tus clases. Ayudan a:

  • Evitar conflictos de nombres (dos clases pueden tener el mismo nombre si están en paquetes diferentes)
  • Organizar el código en módulos lógicos
  • Controlar el acceso a las clases

Ejemplo: com.miempresa.proyecto indica que esta clase está en la carpeta com/miempresa/proyecto

2. Importaciones (import)

Cuando quieres usar clases de otros paquetes, debes importarlas. Por ejemplo:

  • import java.util.ArrayList; - Importa solo la clase ArrayList
  • import java.util.*; - Importa todas las clases del paquete java.util

Nota: El paquete java.lang se importa automáticamente, por eso no necesitas importar System o String.

3. Clase

Es el contenedor principal de tu código. Define un tipo de objeto con sus propiedades y comportamientos.

4. Atributos

Variables que almacenan el estado del objeto. Generalmente se declaran como private para proteger los datos (encapsulación).

5. Constructor

Método especial que se ejecuta al crear un objeto. Su nombre debe ser igual al de la clase y no tiene tipo de retorno.

6. Métodos

Definen el comportamiento del objeto. Son las acciones que puede realizar.

7. Método main

El punto de entrada del programa. Es donde comienza la ejecución.

Convenciones de Nombres en Java

  • Clases: PascalCase (primera letra mayúscula) - MiClase, PersonaJoven
  • Métodos y variables: camelCase (primera letra minúscula) - miMetodo, nombreCompleto
  • Constantes: MAYÚSCULAS con guiones bajos - MAX_VALOR, PI
  • Paquetes: Todo en minúsculas - com.empresa.proyecto
