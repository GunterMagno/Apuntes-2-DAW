5. Secrets y Seguridad

Gestión de credenciales

¿Qué son los Secrets?

Los secrets son variables cifradas para guardar información sensible (API keys, tokens, contraseñas).

⚠️ Nunca expongas secrets en el código!

Crear Secrets

Repositorio → Settings → Secrets and variables → Actions → New repository secret

  1. Name: API_KEY
  2. Value: tu clave secreta
  3. Add secret

Usar Secrets en Workflows

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Deploy
        env:
          API_KEY: ${{ secrets.API_KEY }}
          DB_PASSWORD: ${{ secrets.DB_PASSWORD }}
        run: |
          echo "Deploying..."
          # Los secrets están disponibles como variables de entorno

Variables de Entorno

env:
  NODE_ENV: production
  
jobs:
  build:
    env:
      BUILD_DIR: ./dist
    steps:
      - run: echo $NODE_ENV
      - run: echo $BUILD_DIR
      - run: echo ${{ secrets.TOKEN }}
Secrets predefinidos:
  • GITHUB_TOKEN - Token automático para el repositorio
  • GITHUB_ACTOR - Usuario que inició el workflow
  • GITHUB_REPOSITORY - Nombre del repo
